  5.2. TEMEL TASARIM DESENLERİ

    • Singleton benzeri Küresel Durum:
      - ai_client.cpp'de static değişkenler (API anahtarları, model adları)

    • Tembel Yükleme (Lazy Initialization):
      - page_utils.cpp: ensure_page_built() ile her sayfa ilk
        ziyarette oluşturulur

    • Gözlemci (Observer / Signal-Slot):
      - GTK sinyalleri (signal_clicked, signal_row_selected)
      - Glib::Dispatcher ile thread → UI iletişimi

    • İş Parçacığı (std::thread):
      - AI API çağrıları ayrı thread'lerde
      - Not paylaşımında HTTP istekleri

    • Veri Saklama (Repository Pattern):
      - data_manager.cpp: tüm veri yükleme/kaydetme işlemleri
        tek bir sınıfta toplanmış

    • SM-2 Algoritması (Spaced Repetition):
      - Flashcards modülünde uzun aralıklı tekrar

    • CRUD:
      - Notlar, görevler, flashcard'lar, ders programı için
        oluşturma/okuma/güncelleme/silme işlemleri

  5.3. VERİ AKIŞI

    Kullanıcı Etkileşimi
         │
         ▼
    UI Sinyali (GTK callback)
         │
         ▼
    İşlem (hesaplama/doğrulama)
         │
         ▼
    Veri Modeli Güncelleme (vector, map)
         │
         ▼
    save_data() → .dat dosyaları (XDG)
